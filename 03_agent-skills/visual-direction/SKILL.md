# Visual Direction — Workflow Skill

## Purpose
Define and document the visual strategy for a project or campaign so that all design and development work is governed by a clear, brand-aligned direction brief. This skill produces direction — not final assets or code.

## Use When
- A project requires a visual direction brief before design or development begins
- Visual assets (images, backgrounds, logos, iconography) need strategic alignment with brand rules
- A new UI surface or branding direction is being introduced
- DAA, FIS, or a human designer needs a scoped visual brief with hierarchy, mood, and asset map
- A brand consistency audit is required across existing visual assets

This skill sets visual strategy and produces direction briefs. It does not produce final rendered assets, illustrations, or code.

## Required Inputs
- Project or campaign brief defining the visual need
- Brand identity framework and approved asset library (from client workspace `02_discovery/` or `NoDrftSystems Design System/`)
- Target audience and platform constraints
- Package tier (T1–T5 — determines visual complexity ceiling)
- Accessibility and performance requirements for visual media

## Workflow
1. Confirm the visual need is scoped to a specific deliverable, campaign, or system update
2. Verify that the brand identity framework is on file — if missing: stop and escalate to Founder before any visual work begins
3. Review the existing asset library and platform constraints
4. Define the visual direction statement: mood, hierarchy, motion principles (if applicable)
5. Identify required asset types and specifications (resolution, format, source, licensing)
6. Create the detailed visual brief with layout guidance, color logic, typography rules, and asset map
7. Flag any asset gaps or licensing risks
8. Define visual constraints — what is NOT permitted based on tier scope and brand governance
9. Route the visual brief to DAA, FIS, or the assigned human designer for implementation
10. Maintain a visual system changelog when updates are approved

## Outputs
- Visual direction brief: mood, hierarchy, platform specifications
- Asset requirement list: format, resolution, source, licensing notes for each required asset
- Visual constraints document: what is out of scope or prohibited for this project
- Brand alignment review report (if auditing existing assets): consistency pass/fail with gap list
- Visual system changelog: what changed from prior direction if updating an existing system
- Gap flags: missing, inconsistent, or at-risk visual components with corrective action recommendations

## Block Conditions
- Brand identity framework not on file: stop — do not produce visual direction without it
- Asset licensing risk identified: route to LCA (legal compliance) before the asset is specified for production use
- Client-specific visual requirements violate accessibility standards: flag as IMPORTANT, present compliant alternatives

## Do Not Do
- Do not produce final rendered images, illustrations, code, or deployed assets
- Do not override BCA (Brand Consistency Agent) brand rulings without a documented exception
- Do not approve asset licenses or procurement directly
- Do not create visual briefs without the approved brand identity framework

## Escalation → MOA → HHC when
- Visual direction conflicts with approved brand governance
- A licensing or IP risk requires legal review
- Asset procurement requires budget authority beyond the project owner

**Human authority:** ARE / Assigned human designer
